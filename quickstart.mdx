<Steps>
  <Step title="Prerequisites">
    Before you begin, make sure you have the following:

    - **Python 3.10+** — required by the harness runtime
    - **[uv](https://docs.astral.sh/uv/)** — used to install dependencies and run the CLI
    - **Node.js 18+** — optional, only needed for the React terminal UI
    - **An LLM API key** — Anthropic by default; see [Provider Compatibility](/provider-compatibility) for alternatives

    <Note>
      You can use a Kimi/Moonshot key instead of an Anthropic key. Set
      `ANTHROPIC_BASE_URL=https://api.moonshot.cn/anthropic` and
      `ANTHROPIC_MODEL=kimi-k2.5` alongside your Kimi API key and OpenHarness
      will route requests through the Moonshot Anthropic-compatible endpoint.
    </Note>
  </Step>

  <Step title="Install">
    Clone the repository and install all dependencies with `uv sync`:

    ```bash
    git clone https://github.com/HKUDS/OpenHarness.git
    cd OpenHarness
    uv sync --extra dev
    ```

    This installs the `oh` CLI entry point along with all runtime and development dependencies.
  </Step>

  <Step title="Set your API key">
    Export your API key as an environment variable:

    ```bash
    export ANTHROPIC_API_KEY=your_api_key_here
    ```

    You can also write the key to `~/.openharness/settings.json` under the `api_key` field if you prefer a persistent configuration over a shell export.
  </Step>

  <Step title="Launch OpenHarness">
    Start the interactive agent session:

    <CodeGroup>

    ```bash with venv activated
    oh
    ```

    ```bash without activating venv
    uv run oh
    ```

    </CodeGroup>

    <Tip>
      `uv run oh` invokes the CLI directly through `uv` without requiring you to
      activate the virtual environment first. This works from any shell and is the
      recommended approach when scripting or running one-off commands.
    </Tip>
  </Step>
</Steps>

## One-command demo

Run a single prompt against the current directory without entering the interactive session:

```bash
ANTHROPIC_API_KEY=your_key uv run oh -p "Inspect this repository and list the top 3 refactors"
```

This is a good first test to verify your installation and API key are working.

## Non-interactive mode

Pass `-p` / `--print` to send a single prompt and write the result to stdout. Combine with `--output-format` for structured output:

```bash
# Plain text — default for human-readable output
oh -p "Explain this codebase"

# JSON — structured result for programmatic consumption
oh -p "List all functions in main.py" --output-format json

# Stream JSON — newline-delimited JSON events emitted in real time
oh -p "Fix the bug" --output-format stream-json
```

`stream-json` is useful when piping output into another process that should start processing before the agent finishes.

## Session flags

Control session continuity and naming with these flags:

| Flag | Short | Description |
|------|-------|-------------|
| `--continue` | `-c` | Continue the most recent session |
| `--resume` | `-r` | Pick a past session to resume from a list |
| `--name` | `-n` | Assign a name to the current session |

```bash
# Continue the last session
oh -c

# Resume a named session interactively
oh -r

# Start a new named session
oh -n "refactor-auth-module"
```
